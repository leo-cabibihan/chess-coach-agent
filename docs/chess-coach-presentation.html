AI Engineering Buildcamp Capstone

From 2,775 games to deliberate chess practice.

A grounded learning system that imports a player's full history, finds recurring weaknesses with Stockfish, turns real mistakes into board drills, grades attempts, and adapts review over time. This page maps every major part of the codebase to the techniques taught in the learn-agents course notes.

REST endpoints for game sync, player profiles, training sessions, quiz attempts, and progress. _process_sync_job fetches platform PGNs, skips already-analyzed external IDs, runs Stockfish analysis, and persists results.

Converts centipawn swings into winning-chance loss, classifies inaccuracy/mistake/blunder, and attaches heuristic themes (loose piece, forcing tactics, king safety, opening drift, endgame conversion). No fixed moment quota.

Builds adaptive plans from persisted critical moments. evaluate_attempt checks legality, compares pawn loss to best move, records attempts, and updates review schedule (1d wrong, 3d with hint, 7d clean, 30d mastered).

Title, BM25, vector cosine, and reciprocal-rank-fusion retrieval over lesson chunks. production_strategy() picks BM25 unless hybrid clears explicit MRR and latency gates from the 20-query benchmark.

End-to-end flows

Three pipelines the product actually runs.

1. Full-history sync

React
POST /api/games/sync with platform, username, max games (up to 5,000).
FastAPI
Create or resume one active SyncJobRow; schedule _process_sync_job as BackgroundTask.
importers.py
Fetch Chess.com monthly archives or Lichess PGN stream; preview_pgn_games normalizes metadata.
repositories
Skip external game IDs already analyzed at current detector version.
analysis.py
Replay each new game with reused Stockfish process; persist all qualified moments.
React
Poll GET /api/games/sync/{job_id}; refresh dashboard, library, practice queries.
Review
P1 debt: BackgroundTasks can strand a durable row after process restart. Needs lease/heartbeat worker.

2. Practice session

PracticePage
POST /api/training/sessions — ranks moments by severity, theme accuracy, due date.
training.py
build_training_session adapts move-choice count to player rating and theme performance.
Player
Submits SAN/UCI move via POST /api/training/sessions/{id}/attempts.
engine.py
evaluate_attempt validates legality, measures pawn loss vs best line, returns evaluation panel.
repositories
Updates QuizAttemptRow, ReviewScheduleRow, recomputes player memory.

3. Moment detection (Lichess model)

winning_chance(cp) = sigmoid(cp) loss = chance_before - chance_after accuracy = 103.1668·exp(-0.04354·loss%) - 3.1669 Thresholds (player who moved): ≥ 0.10 inaccuracy ≥ 0.20 mistake (training default) ≥ 0.30 blunder (highest priority)

Heuristics label a Stockfish-verified loss — they never invent a moment. A game can produce zero moments or twenty-five. See docs/moment_selection.md and Lichess analysis references.

Design principle

Use AI to organize teaching experiments. Use chess evidence to decide what is true. Every drill comes from a real game, every answer can be graded, and every attempt changes what the player sees next.
